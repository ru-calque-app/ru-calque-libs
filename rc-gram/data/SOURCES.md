# Источник инвентаря `egp.jsonl`

Файл собран `tools/build_egp.py` из **English Grammar Profile Online** — выгрузки
`.xlsx` с [englishprofile.org](https://www.englishprofile.org/english-grammar-profile).

| Что даёт | Объём |
|---|---|
| «Can-do» описания грамматических конструкций с уровнем CEFR | 1222 записи |
| Разбиение на категории / подкатегории | 19 / 91 |
| Примеры из Cambridge Learner Corpus с уровнем, L1 автора и вердиктом Pass/Fail | 3610 примеров |

Уровни выведены не экспертно, а из Cambridge Learner Corpus: конструкции приписан уровень,
на котором она впервые надёжно появляется в сданных экзаменационных работах. Это главная
ценность файла — калибровка эмпирическая, а не «на глаз».

## ⚠️ Лицензия — открытый вопрос, решать до релиза

**English Grammar Profile — © Cambridge University Press & Assessment.** Сайт даёт
свободный доступ «for research and educational purposes»; явного разрешения на
использование в коммерческом продукте там нет.

Это ровно тот случай, который соседний `rc-lex` уже разбирал и решил в другую сторону: в
`rc-lex/data/SOURCES.md` **English Vocabulary Profile отвергнут** как проприетарный
(«обходных путей нет, лицензию надо покупать у Cambridge»). EGP — его грамматический
близнец из того же проекта English Profile и того же правообладателя. Держать в одном
воркспейсе отказ от EVP и молчаливое использование EGP — непоследовательно.

Пока вопрос не закрыт, крейт годится для исследования и калибровки (сравнить наши
`cefr_min` с корпусными, посмотреть, что вообще бывает на уровне) — но не для выкатки
наружу. Возможные исходы:

1. запросить у Cambridge условия — как и предполагалось для EVP;
2. использовать EGP только офлайн, как инструмент калибровки: в продукт уезжают наши
   собственные `construct_key` с исправленными уровнями, а `egp.jsonl` остаётся в репо
   инструментом разработчика и не линкуется в сервисы;
3. отказаться и калибровать уровни по собственным данным попыток, когда их накопится.

Вариант 2 — самый дешёвый и, скорее всего, достаточный: нам нужны из EGP **уровни**, а не
тексты описаний.

## Пересборка

```bash
pip install openpyxl
python3 tools/build_egp.py ~/Downloads/'English Grammar Profile Online.xlsx'
```

Источник — локальный файл, а не URL: EGP не отдаётся по стабильной ссылке, выгрузка
делается руками с сайта. Результат коммитится и вкомпилирован через `include_str!`.

## Формат

JSONL, по записи на строку. Не TSV (как в `rc-lex`) потому, что примеры многострочные и
содержат кавычки — экранировать их в TSV дороже, чем распарсить JSON.

```json
{"id":"egp:conjunctions.coordinating.c2.02","src_id":"1741163706316x...","level":"C2",
 "category":"CONJUNCTIONS","subcategory":"coordinating","aspect":"form_use",
 "topic":"'AND YET', CONCESSIVE","can_do":"Can use 'And yet' to combine sentences...",
 "lexical_range":2,
 "examples":[{"text":"...","level":"C2","l1":"German","pass":true}]}
```

`id` — наш, читаемый; `src_id` — родной ключ EGP, по нему сверяется пересборка, если
Cambridge поменяет порядок строк в выгрузке.

## Чему в этом файле нельзя доверять

- **`aspect: "use"` (295 записей) не детектируется из текста.** Это описания намерения
  говорящего («чтобы подчеркнуть», «как сигнал завершения»), а не формы. Синтаксически их
  не найти ни сейчас, ни потом. Реально матчибельны только `form` (657) и часть
  `form_use` (264), причём перекос в сторону A1–B1: форма на низких уровнях проще.
- **`lexical_range` есть только у 231 записи** (1/2/3), у остальных `N/A`. Что именно
  означает шкала, EGP толком не документирует — не строить на ней решений.
- **Уровень примера ≠ уровень конструкции.** В примерах к записи C2 встречаются работы
  уровня B2: пример показывает употребление, а не то, что конструкция освоена на B2.
- **`pass: false` (82 примера)** — работа экзамен НЕ сдала, хотя конструкцию содержит.
  Как образец для показа ученику такие примеры не годятся.
- **`l1` — родной язык автора работы, а не язык примера.** Русских L1 — 87 из 3610.
  Как gold-set для калек этого мало, но как срез «что русскоязычные реально произносят на
  каждом уровне» — уже небесполезно.
- **6 записей с `aspect: "other"`** — сломанная разметка в самом источнике
  (`FORM'-ING' FORMS`, `General comment`). Оставлены как есть, чтобы не терять записи.
